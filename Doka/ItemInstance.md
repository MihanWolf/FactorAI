# ItemInstance

**Тип:** Resource  
**Файл:** `Scripts/Item/item_instance.gd`  
**Наследует:** Resource

## Описание
Конкретный экземпляр предмета — радио с царапинами, зелёными проводами и сломанным корпусом. В отличие от [[ItemData]] который один на всех, `ItemInstance` уникален для каждого предмета в мире.

Хранит ссылку на свой чертёж `data`, текущее качество и словарь компонентов в слотах.

## Зависимости
- [[ItemData]] — чертёж этого предмета
- [[ComponentInstance]] — компонент лежащий в слоте
- [[ComponentSlotData]] — описание слота, нужен для проверок

## Сигналы

| Сигнал | Параметры | Когда испускается |
|--------|-----------|-------------------|
| component_changed | slot_id: String | компонент добавлен или снят из слота |

## Переменные

| Имя         | Тип        | Описание                                          |
| ----------- | ---------- | ------------------------------------------------- |
| data        | ItemData   | чертёж этого предмета                             |
| quality     | String     | текущее качество ("Новый", "Старый" и т.д.)       |
| description | String     | описание                                          |
| components  | Dictionary | слот_id → ComponentInstance (или null если пусто) |

## Методы

### create_random(from_data, component_pool) -> ItemInstance
**Статический метод** — создаёт новый экземпляр предмета со случайными параметрами. Вызывается конвейером при спавне предмета.

Как работает:
1. Создаёт новый `ItemInstance` и присваивает ему чертёж `from_data`
2. Выбирает случайное качество из `from_data.quality_levels`
3. Для каждого слота в чертеже ищет подходящие компоненты в `component_pool`
4. Если слот необязательный (`required = false`) — с шансом 20% оставляет его пустым
5. Кладёт случайный компонент в каждый слот

`component_pool` — это словарь `ComponentType → Array[ComponentData]`, строится через `ComponentDatabase.build_pool()`.

---

### place_component(slot_id, component) -> bool
Кладёт компонент в слот. Проверяет что слот существует и что тип компонента подходит для этого слота. Испускает `component_changed`. Возвращает `false` если что-то не так.

---

### remove_component(slot_id) -> ComponentInstance
Вынимает компонент из слота, обнуляет его место и испускает `component_changed`. Возвращает вынутый компонент — его нужно куда-то положить (инвентарь, стол).

---

### is_functional() -> bool
Проверяет работоспособность предмета — все ли обязательные слоты (`required = true`) заполнены. Если хоть один обязательный слот пуст — предмет не работает.

---

### get_slots_by_layer() -> Array
Возвращает слоты предмета отсортированные от верхнего слоя к нижнему. Нужно для стола разборки — чтобы показывать компоненты в правильном порядке снятия.

---

### is_slot_blocked(slot_id) -> bool
Проверяет заблокирован ли слот компонентом сверху. Слот заблокирован если существует другой слот с более высоким `sprite_layer_index` у которого есть компонент. Используется столом разборки — нельзя снять плату пока не снята крышка.

---

### _get_slot(slot_id) -> ComponentSlotData
**Приватный вспомогательный метод.** Ищет и возвращает `ComponentSlotData` по его `slot_id`. Возвращает `null` если слот не найден.
