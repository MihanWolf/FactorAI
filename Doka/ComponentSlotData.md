# ComponentSlotData

**Тип:** Resource  
**Файл:** `component_slot_data.gd`  
**Наследует:** Resource

## Описание
Описание гнезда внутри предмета. Радио имеет гнездо для корпуса и гнездо для проводов — каждое из них это `ComponentSlotData`. Хранится прямо внутри [[ItemData]] в массиве `component_slots`.

Слот описывает **что** туда можно положить и **как** это выглядит в стеке слоёв. Что именно там лежит в конкретном предмете — это уже [[ItemInstance]].

## Зависимости
- [[ComponentData]] — ссылается на его enum `ComponentType`

## Переменные

| Имя | Тип | Описание |
|-----|-----|----------|
| slot_id | String | уникальный ключ слота внутри предмета, например "chassis" или "wire" |
| slot_label | String | читаемое название для UI, например "Корпус" |
| accepted_types | Array[ComponentData.ComponentType] | какие типы компонентов можно сюда положить |
| required | bool | если `true` — без этого компонента предмет не работает |
| sprite_layer_index | int | порядок слоя при отрисовке. Чем выше число — тем выше слой. Компоненты с более высоким индексом перекрывают нижние и блокируют к ним доступ при разборке |

## Как работает sprite_layer_index

```
Радио (вид сверху):
  layer 0 — chassis  (корпус)     ← снизу, доступен всегда
  layer 1 — circuit  (плата)      ← лежит на корпусе
  layer 2 — cover    (крышка)     ← сверху, перекрывает плату

Чтобы снять плату — нужно сначала снять крышку.
Проверяется через ItemInstance.is_slot_blocked()
```
